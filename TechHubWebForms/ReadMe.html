⚙️ Setup & Configuration Guide

Prerequisites

  • ✓ Windows 10/11
  • ✓ Visual Studio 2019/2022 (Community/Professional/Enterprise)
  • ✓ SQL Server Express LocalDB (included with Visual Studio)
  • ✓ .NET Framework 4.7.2 or higher
  • ✓ Modern web browser (Chrome, Edge, Firefox)

Step 1: Extract Project Files

  1. Extract the ZIP file to a location on your computer (e.g., C:\Projects\TechHubWebForms)
  2. Make sure the path doesn't contain special characters or spaces
  3. Ensure all files are unblocked (Right-click → Properties → Unblock)

Step 2: Open Solution in Visual Studio

  1. Launch Visual Studio
  2. Click File → Open → Project/Solution
  3. Navigate to the extracted folder
  4. Select TechHubWebForms.sln
  5. Click Open

Step 3: Restore NuGet Packages

  1. In Solution Explorer, right-click on the solution
  2. Select Restore NuGet Packages
  3. Wait for all packages to download and install
  4. Required packages: EntityFramework 6.x, Bootstrap, jQuery

Step 4: Database Configuration

Good News: The database is automatically created on first run using Code-First migrations!

Connection String Location: Web.config

<connectionStrings> <add name="TechHubConnection" connectionString="Data Source=(LocalDb)\MSSQLLocalDB; Initial Catalog=TechHub; Integrated Security=True" providerName="System.Data.SqlClient" /> </connectionStrings>
⚠️ Note: If database creation fails, check if SQL Server LocalDB is installed. Run sqllocaldb info in Command Prompt to verify.

Step 5: Build the Solution

  1. Press Ctrl + Shift + B or click Build → Build Solution
  2. Check the Output window for any errors
  3. All errors should be resolved before running

Step 6: Run the Application

  1. Press F5 or click the green Play button (IIS Express)
  2. Wait for the browser to open automatically
  3. Default URL: http://localhost:XXXXX/
  4. The home page should load with product categories
✓ Success! If you see the TechHub home page, the setup is complete!

🔧 Troubleshooting

Issue: Database Not Created

Solution: 1. Check SQL Server LocalDB is installed: sqllocaldb info 2. Delete existing database: sqllocaldb delete MSSQLLocalDB 3. Rebuild solution and run again

Issue: Build Errors

Solution: 1. Clean Solution: Build → Clean Solution 2. Restore NuGet Packages: Right-click solution → Restore NuGet Packages 3. Rebuild: Build → Rebuild Solution

Issue: 404 Not Found

Solution: 1. Check URL is correct (e.g., /Default.aspx, not /Home) 2. Verify file exists in Solution Explorer 3. Check file properties: Build Action = Content

Issue: Images Not Showing

Solution: 1. Check Content/Images/Products/ folder exists 2. Upload images via Admin → Manage Products 3. Verify image URLs in database are correct
